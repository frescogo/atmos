# 260717 - Radar Integration

Integrate the serial speed radar into the Atmos scoreboard.
The radar work already exists in `../ceu/`; the scoreboard
(`../atmos/`) still has `conf.radar()` hardcoded to `false`.

## Steps

- [ ] 1. Copy `radar.atm` from `../ceu/` into `../atmos/`
    - adapt env: ceu uses `atmos.env.sdl`, atmos uses
      `atmos.env.pico` (drop the require; env comes from main)
    - replace `require "global"` (G.reps) with `CONF.radar.reps`
- [ ] 2. Fix known issues in copied `radar.atm`
    - typo `:radaw.zer` -> `:radar.vel.zer` (Shots watching)
    - `string.sub(s,8)` -> `string.sub(s,8,8)` in dir byte read
    - stray indentation in the `jog` ifs
- [ ] 3. `main.atm`: spawn `Probe(dt)` + `Shots()`;
      `conf.radar()` = any radar task alive (track via
      `:radar.on` true/false)
- [ ] 4. `partida.atm`: radar path alongside keyboard path
    - saque: `:radar.shot` with `vel >= CONF.saque` starts rally
    - golpe: `:radar.shot` appends
      `[now=..., jog=..., kmh=vel]` to `REGRA.seqs`
      (jog from shot side, `CONF.lado.radar` maps dir->side)
    - auto mode: timeout without shots -> queda
      (placar.pde `RADAR_AUTO_TIMEOUT` = 3500ms)
- [ ] 5. Decide filter semantics: ceu `Shots()` (4 reps, 25%
      gap, 250ms cooldown) vs placar.pde `radar_be()`
      (igual=700ms / oposi=300ms direction timing);
      `conf.atm` carries both parameter sets
- [ ] 6. `tela.atm`: radar ok/no icon on `CONF.lado.radar` side
      (placar.pde: `RADAR_ERR <= 10`)
- [ ] 7. `conf.atm`: honor `serial` field
      ("" = auto-probe, "desligado" = off, or explicit dev)

## Notes

- Protocol: 23-byte frame, sync `0x88`; dir byte at offset 8
  (66 = inb, 64 = out); speed = ascii digits at offsets 9-12,
  rounded /10 to km/h.
- `regra.atm` `M.kmh` already prefers `cur.kmh` when present,
  so scoring needs no changes (radar golpes carry kmh; manual
  golpes derive from time between golpes).
- Simulator: flash `../ceu/arduino/radar/radar.ino`
  (always sends; random dir; 50% vel=0; else 30-89 km/h).
