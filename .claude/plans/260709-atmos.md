# Port placar.pde to pico-atmos

**Reference switched (2026-07-10) from v5.3.0 to v4.3.0**
(branch `old` @ 1af7111 in ../placar-radar): conf uses
`ataques`/`aborta` (no golpes/continuos/potentes/prefixo),
scoring `kmh^2/50`, quedas pct penalty (`jogo_quedas_pct`)
+ aborta, no maximas/intensidade/backhands, no blink at 5s,
médias only at termino, `conta` line under TOTAL.
Phase 2-4 checklists below still describe v5.3 features:
drop C-9 (prefixo), C-- guard exists, Z/M backs do not
exist, KEY_TIMER=3s, bE radar same.

Target: `main.atm` at the repo root (`/x/x/frescogo/atmos`),
run with `atmos main.atm` from the root.
Assets: no symlink; copy files one by one from
`../placar-radar/placar/data/` into `./data/` as needed
(paths in code: `data/...`).
Reference idioms: pico-rocks `main.atm`
(https://github.com/atmos-lang/pico-rocks):
`require "atmos.env.pico"`, `loop on :draw`,
`watching`, `:key.dn [key=...]`, `[ '%', x=, y= ]`.

Processing grid `W=width/11`, `H=height/8` maps to `%` coords:
`x = col/11`, `y = row/8`.
Window: `pico.set.dim ['!', w=1000, h=600]`.

## Phase 2: key bindings                                [~]

Done step by step (2026-07-10). Architecture:
- files: `main.atm` (config/state/input/wiring),
  `tela.atm` (LAYOUT + section tasks, returns `Tela`),
  `partida.atm` (returns `Partida` FSM), `conf.atm` (data)
- shared state via `set` globals: pico, CONF, conf, G, WIN,
  NOMES, JOGO (val/var are chunk-local; func/task are
  global by definition)
- `task Hold(key)`: ctrl+key held 1s (v4.3 uses 3s), any
  key.up restarts, peteleco on confirm, returns only after
  ctrl released
- `task Digita(alvo)`: typing with in-string `_` cursor,
  Return keeps / Escape or any ctrl-key restores, defer
  cleans `G.edit` + cursor
- `Partida`: ocioso -> C-Up saque -> Left/Right golpes
  (`JOGO.seqs@#@+`) -> C-Down queda -> ocioso
- bindings composed with `await Hold(..)` + `par:any`
- `pico.set.aids(false)`: new pico-sdl API disabling the
  builtin ctrl debug keys (was swallowing C-0 etc.)
