## File map

| file            | content                                   |
|-----------------|-------------------------------------------|
| `main.atm`      | conf table, globals, Hold/Digita/Undo     |
|                 | tasks, key bindings, game lifecycle loop  |
| `tela.atm`      | LAYOUT (3x3 sections) + one drawing task  |
|                 | per section; returns `Tela`               |
| `partida.atm`   | game FSM; returns `Partida`               |
| `regra.atm`     | facts (seqs/quedas/descanso/now) +        |
|                 | reactive rules; returns module `M`        |
| `radar.atm`     | `Mock` + `Serial` + dispatcher `Radar`    |
| `relatorio.atm` | png/txt/csv writer; returns func          |
| `conf.atm`      | data (from conf.json)                     |

## Architecture invariants

- Cross-chunk state via `set` globals (val/var are
  chunk-local): pico, CONF, REGRA, G, NOMES, conf,
  relatorio. func/task declarations are global by
  definition (`func f` == `set f = func`).
- `G.estado`: `:ocioso | :jogando | :jogando.sacando |
  :terminado` — written ONLY by `Partida` (a `defer` sets
  `:terminado` on any exit, incl. aborts).
- Undo restarts `Partida`: any successful undo aborts and
  respawns it; the fresh instance resumes over the
  persisting `REGRA` in `:ocioso` (see `task Undo` +
  `watching Undo()` in main).
- `jog` = player identity (`:esq/:dir`), `pos` = physical
  side. `inv()` maps screen->player for KEYS ONLY; the
  radar is physical (no inv). The inv swap lives ONLY in
  `Tela`'s respawn loop (`await :inv`).
- Events: `emit @:global :golpe/:queda/:inv/:radar`
  (plain `emit` stays in the emitting task's subtree).
- Reactive rules: `REGRA.jog/equ/total/tempo.*` derive
  everything from `M.seqs` per frame; no cached scores.
- Real numbers internally; `math.floor` only at outputs
  (beware pipe precedence: `a / b -> f` parses as
  `a / (b -> f)`; use `-->` or parens).
- Varying texts: `pico.output.draw.text.dyn(key, s, rect)`
  (stable id, '~' fingerprint); `.fix` for static.

## Notes / caveats (learned)

- pico-sdl upstream fixed: ctrl debug keys
  (`pico.set.aids(false)`), text '~' fingerprint
  (text.dyn/.fix). Still open upstream: accented glyphs
  assert in `_tex_text` ("Arbitro" workaround in conf.atm).
- emit awakes children BEFORE parents (abort/priority);
  `watching <Task()>` and `await <task>` both work;
  `watching/await ... until (cond)` for condition waits;
  `until(c)`/`while(c)` break loops (loop yields the
  breaking value); `break [v]` returns a value; `where`
  blocks attach statements to expressions.
- `val M = []` in a module is private — other chunks must
  go through the global (e.g. `REGRA`), never `M`.
- Undoing does NOT revert descanso (v4.3-faithful).
- Time literals: `1s`, `500ms`, `@.250`; clock event
  `:clock` carries the frame delta (`loop dt on :clock`).
