# 260718-exe — Windows package (self-contained folder + exe)

## Goal

Distribute FrescoGO! as a self-contained Windows folder with a
`frescogo.exe` launcher.
No installation required on the target machine.

## Feasibility

Yes.
The stack is highly portable:

| Component      | What                          | Windows story                 |
|----------------|-------------------------------|-------------------------------|
| Lua 5.4        | interpreter                   | prebuilt `lua54.exe/.dll`     |
| atmos          | pure Lua (compiler + runtime) | copy as-is                    |
| pico (Lua)     | pure Lua wrapper              | copy as-is                    |
| pico_native    | C module over SDL2            | cross-compile w/ mingw-w64    |
| SDL2 family    | SDL2, image, mixer, ttf, gfx  | official/prebuilt DLLs        |
| lua-periphery  | serial radar (`radar.atm`)    | **Linux-only — needs port**   |
| app            | `*.atm` + `data/` (1.7M)      | copy as-is                    |

Not needed: lanes (no `thread` usage), socket, iup.

## Layout of the final folder

```
FrescoGO/
├── frescogo.exe          (launcher)
├── lua54.dll
├── SDL2.dll  SDL2_image.dll  SDL2_mixer.dll
├── SDL2_ttf.dll  SDL2_gfx.dll
├── pico_native.dll
├── lua/                  (atmos/, pico/, argparse, ...)
├── main.atm  conf.atm  regra.atm  partida.atm
├── radar.atm  tela.atm
└── data/                 (icos/, snds/, font, png)
```

## Steps

### 1. Cross-compile toolchain (on Linux)

- [ ] install `mingw-w64` (`x86_64-w64-mingw32-gcc`)
- [ ] download Lua 5.4 win64 binaries (or build from source)
- [ ] download SDL2 / image / mixer / ttf mingw dev packages
- [ ] build/download `SDL2_gfx` for mingw (no official binary)

### 2. Build `pico_native.dll`

- [ ] compile `pico.c` (from pico-sdl sources) with mingw
      against SDL2 dev libs + Lua 5.4 headers
- [ ] verify it loads: `require "pico_native"` on Windows/Wine

### 3. Assemble the folder

- [ ] script `tools/mkwin.sh` in the repo that:
    - copies Lua/SDL2 DLLs
    - copies `atmos/`, `pico/` Lua trees into `lua/`
    - copies `*.atm` and `data/`
- [ ] launcher `frescogo.exe`:
    - tiny C stub (mingw): sets `package.path/cpath` to the
      folder, requires atmos, runs `main.atm`
    - alternative (simpler first pass): `frescogo.bat` calling
      `lua54.exe atmos.lua main.atm`

### 4. Serial radar on Windows

`radar.atm:26` uses `require("periphery").Serial` — Linux-only.
Options (pick one):

| Option        | Description                     | Trade-off        |
|---------------|---------------------------------|------------------|
| luars232      | cross-platform serial C lib     | one more DLL     |
| COM as file   | `mode COMx` + `io.open` raw     | blocking reads   |
| mock only     | Windows uses `Mock` task        | no radar (v1)    |

- [ ] v1: force `Mock` on Windows (detect via `package.config`)
- [ ] v2: port `Serial` to luars232 (nonblocking read w/ timeout
      matches current `f::read(1, 0)` polling)

### 5. Test

- (user) run under Wine on Linux for a smoke test
- (user) run on a real Windows machine: video, fonts, sounds,
  keys, fullscreen

## Single-exe variant (optional, later)

- luastatic/srlua can glue Lua sources into the exe, but
  `pico_native` + SDL2 DLLs would still sit next to it unless
  statically linked (SDL2 static + mingw): *won't do* unless
  the folder version proves insufficient — zip of the folder is
  the practical deliverable.
